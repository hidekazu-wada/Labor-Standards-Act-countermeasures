# Claude Code 指示書

## 正本

このプロジェクトの正本は `PROJECT.md` です。
プロジェクトの目的・フェーズ・進行上の制約・LLM体制・成功指標・撤退方針はすべてそちらを参照してください。

このファイル（CLAUDE.md）は、Claude Codeが自動的に読み込むための派生ビューであり、PROJECT.md と矛盾してはいけません。

---

## このプロジェクトの性質

- **コードを書かないプロジェクト**です。成果物は Markdown ドキュメントと意思決定ログです
- 対象ユーザーは実質1名（プロジェクトオーナーである役員本人）
- フェーズ1: リサーチ＆ドキュメント整備 → フェーズ2: 改善プロジェクト管理

---

## ドキュメント規約

- 日本語で記述する
- 役員・スタッフ・会社の固有名は原則使わない（役職表記に置換）。特に外部LLMに渡す可能性のあるドキュメントでは厳守
- 1つのドキュメントは単一の目的に絞る

---

## LLM役割分担

詳細は PROJECT.md の「LLM役割分担」セクションを参照。要点のみ:

- **Claude Code（オーナー）** — 正本（PROJECT.md, doc/）の更新権限を持つ唯一のLLM。ドキュメント作成・全体管理
- **Perplexity（スカウト）** — 労基法・判例・他社事例のリサーチ。提案までで正本は更新しない
- **ChatGPT（監査役）** — 役員向け資料の第三者レビュー。`doc/evaluation-template.md` + `doc/project-context.md` を渡す

---

## ドキュメント体系

| ファイル | 読者 | 目的 |
|---------|------|------|
| PROJECT.md | オーナー / Claude Code | プロジェクト正本 |
| CLAUDE.md | Claude Code | 自動読み込み指示書（このファイル） |
| doc/project-context.md | 監査役（ChatGPT） | 評価用コンテキスト |
| doc/evaluation-template.md | 監査役（ChatGPT） | 評価依頼テンプレート |

**原則:** 正本は常にgit側。外部LLMやツールのワークスペースは実行環境であり正本ではない。

---

## データガバナンス

詳細は PROJECT.md「データガバナンス」表を参照。要点:

- 会社名・個人名（役員・スタッフ）→ 役職表記に置換、外部LLM非共有
- 社内政治情報（役員間の温度感など）→ 外部LLM非共有
- 実データ（シフト・打刻）→ 匿名化してから扱う
- 労基法一般論・判例・他社事例・仮名化された課題構造 → 外部共有OK

---

## 進め方の原則（このプロジェクト固有）

役員合意形成が完了するまでの間、以下を常に意識する:

- 「違反の指摘」より「リスク管理・人材定着」など前向きなフレーミングを優先
- 一気に全体改革ではなく段階的な合意形成
- 役員間の摩擦を避ける選択肢を常に用意する

---

## 重要な約束事

- **記録を残す:** 見送りも含めて全ての判断を PROJECT.md の「重要な設計判断の履歴」に追記する
- **撤退条件はチェックポイント方式:** 数値で切らず、CP1〜CP3 で立ち止まって判断する（PROJECT.md 参照）
- **ドキュメントの整合性:** PROJECT.md と doc/project-context.md は同期させる。重要な変更時は両方を確認
- **コミットは指示があったときだけ:** 勝手にコミットしない
