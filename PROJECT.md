# 労務コンプライアンス改善プロジェクト

> このファイルはプロジェクトの**正本**です。目的・フェーズ・技術スタック・運用手順・設計判断はすべてここに集約します。

---

## プロジェクト概要

**目的:** 会社の労務運用における労働基準法違反（①変動シフト制の事前周知不足、②シフト表への休憩時間未明記、③タイムカードの1分単位非対応）を解消し、新規入社者も安心して働ける労務体制を整える。

**対象ユーザー:** 実質1名（プロジェクトオーナーである役員本人）。経営者・労務担当・現場スタッフは"受益者"であって、本ドキュメントの直接の読者ではない。

**現在のフェーズ:** フェーズ1 — リサーチ＆ドキュメント整備（法令論点整理・現状ギャップ分析・役員向け説明資料の作成）。社労士・社内担当者への事実確認を依頼済み、回答待ち

**フェーズ構成:**
- **フェーズ0:** 役員合意形成（後述「進行上の制約」参照／時限セクション）
- **フェーズ1:** リサーチ＆ドキュメント中心 — 法令整理、ギャップ分析、役員向け説明資料
- **フェーズ2:** 改善プロジェクト管理 — 課題リスト・進捗・意思決定ログの運用

**業務上の前提:**
- ゲストのスケジュール都合でシフト確定が直前になりやすい業態
- 現場担当者レベルでは改善が進まず、会社レベルでの対応が必要

## 進行上の制約（フェーズ0: 役員合意形成）

> **注意:** このセクションは時限的です。役員合意形成が完了した時点で削除されます。

- 本件はスタッフからの相談がプロジェクトオーナー個人に届いたことが発端であり、**他の役員はこの問題提起自体をまだ知らない**
- 内容が「現状の労務運用は労基法違反」という前向きに受け止めにくいものであるため、切り出し方を誤ると役員間の摩擦になり、改善自体が頓挫するリスクがある
- 進め方の原則:
  - 「違反の指摘」より「リスク管理・人材定着」など前向きなフレーミングを優先する
  - 一気に全体改革を進めるのではなく、役員合意を段階的に積み上げる
  - スタッフ・役員の固有名や役員間の温度感などの社内政治情報は外部LLMに渡さない

## LLM役割分担

| 役割 | 担当 | 責任範囲 | 呼び出し条件 |
|------|------|---------|------------|
| オーナー | Claude Code | 全体管理・正本更新・ドキュメント作成 | 毎セッション |
| スカウト | Perplexity（Comet/Computer） | 労基法・判例・他社事例のリサーチ | 論点整理・事例収集が必要なとき |
| 監査役 | ChatGPT | 役員向け資料の第三者レビュー、大きな判断のセカンドオピニオン | フェーズ移行前・対外資料の確定前 |

**原則:**
- 正本を更新できるのはオーナー（Claude Code）だけ
- スカウトは提案のみ、監査役は評価のみ
- LLM間のデータ受け渡しは人間が仲介

### データガバナンス

| 区分 | ルール |
|------|--------|
| 会社名・個人名（役員・スタッフ） | 外部LLMに渡さない。役職表記に置換する |
| 社内政治情報（役員間の温度感など） | 外部LLMに渡さない |
| 実データ（シフト表・タイムカード等） | 扱う場合は匿名化する（個人名・日付を除去） |
| 労基法の一般論・判例・他社事例 | 外部LLM共有OK |
| 仮名化された課題構造 | 外部LLM共有OK |

## 撤退条件・成功指標

### 成功指標（何をもって「勝ち」とするか）

(A)(B)(C) を段階的マイルストーンとしてすべて達成することを目指す。

| # | 指標 | 内容 |
|---|------|------|
| A | 役員合意 | 他役員が問題を認識し、改善方針に合意している（フェーズ1のゴール） |
| B | 実運用改善 | シフト事前周知・休憩明記・1分単位打刻の3点が実運用レベルで改善されている（フェーズ2のゴール） |
| C | スタッフの実感 | 相談を上げたスタッフが「会社として動いてくれた」と実感できる状態（A+Bの自然な結果） |

### 撤退条件（チェックポイント方式）

数値での撤退判断は本案件には馴染まないため、**事前に撤退条件を決めず、立ち止まって判断するチェックポイントだけ置く**。

| # | チェックポイント | 判断内容 |
|---|---------------|---------|
| CP1 | 最初の役員打診を終えた時点 | 反応を見て、継続/修正/保留のいずれかに振り分ける |
| CP2 | 役員会など公式な場で議題化した時点 | 合意形成の見通しを再評価 |
| CP3 | フェーズ1（ドキュメント整備）完了時点 | フェーズ2に進むか、一旦止めるか判断 |

### 切り替えオプション（撤退ではなく方向転換）

- **社内ルートが閉ざされた場合** → スタッフへの誠実な説明 + 外部ルート（労基署・社労士相談）の案内に切り替える
- **オーナー自身の役員としての立場が危うくなる兆候が出た場合** → プロジェクトを一旦凍結し、時期を改める
